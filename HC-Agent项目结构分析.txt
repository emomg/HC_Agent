一、项目概览
================================================================================

项目名称：HC-Agent (Hybrid Context Agent)
项目路径：D:\桌面\HC-Agent\
代码总量：约 90 KB (15 个 Python 源文件 + 辅助资源)
设计哲学：受 DeepSeek V4 论文启发，CSA+HCA 混合注意力记忆 + CDH 上下文预算
         + 论文进化系统 + 10 轮反思引擎，原生 ReAct 推理循环

核心创新点：
  [1] CSA (Contextual Semantic Attention) -- 语义相关性评分
  [2] HCA (Historical Context Attention) -- 历史注意力评分
  [3] CDH (Character-Domain Heuristic) -- 字符域启发式上下文预算分配
  [4] 论文进化系统 -- 自动搜集论文 -> 提取技能 -> 权重升级
  [5] 10 轮反思引擎 -- 历史压缩、技能分析合并、记忆优化

  4.4 memory/store.py -- 记忆存储 (302 行)
  -----------------------------------------------------------------------
  职责：CSA+HCA 混合注意力记忆系统

  四层记忆结构：
    L0 (Raw)      原始对话记录，自动存储
    L1 (Index)    索引摘要，快速检索
    L2 (Summary)  压缩摘要，反思生成
    L3 (Skill)    技能条目，权重管理

  MemoryItem 数据结构：
    id: str              唯一标识 (SHA1 前 8 位)
    content: str         内容文本
    layer: int           层级 (0-3)
    tags: list[str]      标签列表
    domain: str          知识域
    created_at: float    创建时间戳
    last_accessed: float 最后访问时间戳
    access_count: int    累计访问次数
    source: str          来源 ("user"/"tool"/"paper")
    importance: float    重要性 [0.0, 1.0]

  CSA 评分公式：
    score_csa = keyword_relevance * alpha
              + recency_decay(t) * beta
              + log(1 + access_count) * gamma

    其中：
      keyword_relevance = TF-IDF 匹配度
      recency_decay(t)  = exp(-t / decay_hours)
      alpha=0.6, beta=0.3, gamma=0.1

  HCA 评分公式：
    score_hca = layer_priority * access_recency * compression_bonus

    其中：
      layer_priority    = [1.0, 0.8, 0.6, 0.4] (L0-L3)
      access_recency    = exp(-time_since_access / 6h)
      compression_bonus = 1.0 + 0.1 * (max_layers - layer)

  混合分数：
    final_score = w_csa * score_csa + w_hca * score_hca
    (w_csa = 0.6, w_hca = 0.4 默认)

  核心方法：
    add(content, layer, tags, domain, source, importance)
    search(query, top_k, domain_filter, layer_filter)
    compress(layer)  -- HCA 层间压缩
    get_all(layer)   -- 获取某层全部条目
    get_stats()      -- 统计信息


  4.5 memory/budget.py -- CDH 预算分配器 (150 行)
  -----------------------------------------------------------------------
  职责：将有限的上下文窗口分配给不同记忆类别

  CDH 公式：
    score_i = relevance_i^alpha * recency_i^beta * importance_i^gamma
    budget_i = total_budget * score_i / sum(scores)

  分区分配：
    working  (40%)  工作记忆 -- 当前任务直接相关
    skills   (25%)  技能记忆 -- SOP/技能条目
    facts    (15%)  事实记忆 -- 知识条目
    history  (20%)  历史记录 -- 对话历史

  域匹配加成：
    如果条目 domain 与当前查询 domain 匹配，
    其预算分配乘以 cdh_domain_boost (默认 1.5x)

  核心方法：
    allocate(query, store, history_text, total_budget)
      -> BudgetAllocation(working, skills, facts, history)

  返回结构：
    BudgetAllocation:
      working: list[MemoryItem]   工作上下文条目
      skills: list[MemoryItem]    技能上下文条目
      facts: list[MemoryItem]     事实上下文条目
      history: list[str]          历史文本片段
      total_tokens: int           已分配总 token
      remaining: int              剩余预算

  4.7 agent_loop.py -- ReAct 推理循环 (277 行)
  -----------------------------------------------------------------------
  职责：实现 ReAct (Reason+Act) 推理循环

  五步循环：
    Step 1: Perceive
      - 调用 CDHBudgetManager.allocate() 构建上下文
      - 注入工作记忆、技能、事实、历史

    Step 2: Think (Inner Monologue)
      - 构建 prompt 包含：系统提示 + 上下文 + 用户输入
      - 调用 LLM 进行推理

    Step 3: Act
      - 解析 LLM 输出，提取 tool_call
      - 路由到 ToolRegistry 执行

    Step 4: Observe
      - 收集工具执行结果
      - 存入记忆系统 (L0)

    Step 5: Reflect
      - 每 10 轮触发 ReflectionEngine
      - 压缩历史、分析技能、优化记忆

  Inner Monologue 输出格式：
    <think>
    [推理过程]
    </think>
    <tool>
    {"name": "tool_name", "args": {...}}
    </tool>
    -- 或 --
    <answer>
    [最终回答]
    </answer>

  TurnRecord 数据结构：
    turn_id: int
    user_input: str
    agent_output: str
    tool_calls: list[dict]
    tool_results: list[dict]
    context_used: BudgetAllocation
    timestamp: float

  4.10 evolution/skill_upgrader.py -- 技能升级器 (186 行)
  -----------------------------------------------------------------------
  职责：技能生命周期管理

  技能生命周期：
    Discovery -> Creation -> Usage -> Upgrade -> Pruning

  权重更新公式：
    new_weight = old_weight * DECAY_RATE
               + usage_count * USAGE_BOOST
               + paper_validation * PAPER_BOOST

  关键常量：
    USAGE_BOOST     = 0.05    每次成功使用加成
    PAPER_BOOST     = 0.10    论文验证加成
    DECAY_RATE      = 0.995   每天未使用衰减率
    MERGE_THRESHOLD = 0.70    技能合并相似度阈值
    PRUNE_THRESHOLD = 0.10    最低保留置信度

  核心方法：
    upgrade_from_papers(topic, domain) -> list[str]
    record_usage(skill_name, success) -> float
    merge_skills(skill_a, skill_b) -> str
    prune_unused(days_threshold) -> list[str]
    get_skill_stats() -> dict


  4.11 evolution/reflection.py -- 反思引擎 (196 行)
  -----------------------------------------------------------------------
  职责：每 10 轮执行三阶段反思优化

  Phase 1: 历史压缩 (History Compression)
    - 将最近 N 轮对话总结为简洁笔记
    - 提取关键决策和结果
    - 存储为 L2 压缩摘要
    - 更新 L1 索引

  Phase 2: 技能分析 (Skill Analysis)
    - 扫描近期工具使用模式
    - 高频成功技能 -> 提升置信度
    - 未使用技能 -> 应用衰减
    - 发现技能缺口 -> 标记为进化候选

  Phase 3: 记忆优化 (Memory Optimization)
    - 合并相似 L0 条目
    - 高访问 L0 条目提升到 L1
    - 老旧 L1 条目压缩到 L2
    - 低重要性条目修剪
    - 重建 L1 索引

  核心方法：
    reflect(turns, store) -> dict    -- 运行完整反思周期
    compress_history(turns) -> str   -- 历史压缩
    analyze_skills(store) -> dict    -- 技能分析
    optimize_memory(store) -> dict   -- 记忆优化

  反思报告结构：
    {
      "reflection_count": 3,
      "turns_processed": 30,
      "history_compressed": 15,
      "skills_boosted": 2,
      "skills_decayed": 1,
      "memory_merged": 3,
      "memory_promoted": 5,

六、设计模式与架构特点
================================================================================

  [1] 分层架构
      表现层 (Console) -> 协调层 (HCAgent) -> 核心层 (Loop/Store/Tools)

  [2] 依赖注入

  [3] 策略模式

      mykey.py 根据 provider 名称自动创建 LLM 配置

  [6] 组合模式
      ToolRegistry 统一管理异构工具，对外提供一致接口


================================================================================
七、关键算法总结
================================================================================

  7.1 CSA 评分
      S_csa = TF_IDF(q, item) * 0.6
            + exp(-age_hours / 24) * 0.3
            + log(1 + access_count) * 0.1

  7.2 HCA 评分
      S_hca = [1.0, 0.8, 0.6, 0.4][layer]
            * exp(-hours_since_access / 6)
            * (1 + 0.1 * (4 - layer))

  7.3 混合分数
      S_final = 0.6 * S_csa + 0.4 * S_hca

  7.4 CDH 预算分配
      budget_i = total * S_final^norm / sum(S_final_all)
      domain_match_bonus = 1.5x

  7.5 技能权重更新
      W_new = W_old * 0.995 + usage * 0.05 + paper * 0.10

  7.6 技能衰减
      W_daily = W * 0.995^(days_since_use)
      prune if W < 0.10
